В примере настроено несколько конфигураций (тег route) для рассылки обновлений в торговые точки из центра. Описание первого маршрута: из очереди Kafka "to_shop" (from uri="jms:queue:to_shop") принимается xml файл-задание на отправку сообщения. В зависимости от тега shopCod используя XPath (//shopCod = '07260') файл отправляется по тому или иному маршруту Camel. Суть других маршрутов примерно понятно из конфигурации Camel. Где:
  • from uri - источник сообщения
  • quartz2 - планировщик
  • bean - Spring bean
  • multicast - оправка сообщения по нескольким маршрутам
